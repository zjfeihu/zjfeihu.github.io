ie下document.write出现浏览器卡死bug

##前言

我们知道，ie下一次性document.write出来的script执行顺序不一定是从上到下的，而为了实现保持执行顺序，就非得采用一些技巧，而这在ie下就碰钉了，就一坑字了得。

##代码

code.html
<iframe name="winbox"></iframe>
<br>
ie下准备任务管理器强制关闭吧。
<button onclick="xopen()">xopen</button>
<script>
function xopen(){
    var doc = window.open('', 'winbox');
    doc.document.write('<script src="../style/z.js"><\/script>');
    doc.document.write('<script>alert(111)<\/script>');
    doc.document.close();
}
</script>
.
info ie下为了保持script执行顺序，需要分开document.write(&lt;script>...);
info ie下当通过多次document.write(&lt;script>...)标签请求外部js文件，浏览器就会卡死在哪里。
