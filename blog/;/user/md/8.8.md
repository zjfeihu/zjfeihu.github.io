js中常见的易错场景

##一些怪异的解释

1. 对象属性包含关键字
2. 对象结尾出现逗号
3. replace的问题

###对象属性包含关键字

执行以下代码，IE下报错了。

code.js
var obj = { default: 'abc' };
alert(obj.default);
.

info 在IE下会出现错误，具体原因？解决方法是关键字属性使用引号括起来。

###对象结尾出现逗号

code.js
var obj = { 'default': 'abc', };
alert(obj['default']);
.

info IE下报错了，具体原因？解决方法是去除末尾的逗号。

###replace的问题

code.js
var str = '价格:$price'; 
alert([
    str.replace(/\$price/, '$_99'), //出现异常
    str.replace(/\$price/, '$+99'), //出现异常
    str.replace(/\$price/, '$99'), //正常
    str.replace('$price', '$+99') //正常
].join('\n'));
.
info IE下返回『价格:价格:$price99』，而不是『价格:$_99』。这个问题产生的原因是IE会把『$_』匹配成『价格:$price』，『$+』匹配成『$price』。

info 在使用replace替换字符串的过程中，请注意替换的内容中是否包含匹配变量『$1-9』,使用$$方式转义$字符串。


##一些易错的场景

1. 获取月份
2. 局部变量泄漏
3. 浮点运算精度问题
4. typeof误用
5. parseInt非10进制转换


###获取月份

code.js
var date = new Date();
alert(date.getMonth());
.

info 获取当前月份必须+1，因为获取到的月份是从0开始的。

###局部变量泄漏

code.js
function fn(){
	var a = b = 12;
}
fn();
alert(typeof a +'\n'+ typeof b); 
.

info 有时一不小心就写出上面var a=b=1的代码，殊不知b已经成全局变量了。

###浮点运算精度问题

code.js
alert(8.20 + 5.10); //13.299999999999999
.

info JS浮点运算会出现精度问题，所以很容易造成一些难以排查的逻辑问题。

###typeof误用

code.js
alert(typeof NaN); //number
.

info 其中还包括Infinity，-Infinity，null等，容易造成疏忽。

###parseInt非10进制转换

code.js
alert(parseInt('077')); //ie8：63，其他：77
alert(parseInt('078')); //ie8：7，其他：78
.

info 在ie8浏览器中测试，出现了非预期的结果，ie8针对0开头的字符串使用8进制进行转换，而非10进制。
