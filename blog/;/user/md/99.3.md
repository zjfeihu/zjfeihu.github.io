WEB++开发之代理模式

##前言

WEB++桌面系统中，采用了观察者模式实现模块间的操作，但是单纯的观察者模式是存在问题的。

##消息的发送和接收

在常规的观察者模式中，消息的发送主体也是消息接听的主体，比如以下

code.js2

张老师 = { 
    接收作业: function(结果){
        处理结果
    }
};

张老师.fire('布置作业', 第一道题, 第三道题);

###小刚同学：

小刚同学 = {
    完成作业: function(第一道题, 第三道题){
        if(第一道题太难){
            return 随便应付，然后跳过第三道题
        }else{
            return 轻松搞定2道题
        }
    }
};

张老师.on('布置作业', function(){
    张老师.接收作业(小刚同学.完成作业())
});

###小毛同学：

小毛同学 = {
    完成作业: function(){
        return 不管三七二十一，认认真真做作业。
    }
};

张老师.on('布置作业', function(){
    张老师.接收作业(小毛同学.完成作业())
});

.

info 张老师作为消息的发起者，也作为被监听者主体，这是常规的观察者模式，但是，其他什么李老师，王老师也要布置作业那该怎么办？

这就像WEB++中各种应用会发起消息，而其他扩展模块会接收消息并处理，比如

code.js2

//桌面中有2个应用，点击应用图标都会向扩展模块发送打开应用的指令
app1 = new App('app1');
app2 = new App('app2');

function App(){
    this.icon.click(function(){
        this.fire('open');
    });
}

//那么其他扩展模块是否需要像以下方式一个个写呢？
app1.on('open', ...)
app2.on('open', ...)

.

info 显然为每个应用添加事件监听那是不可能的事情，也根本办不到， 那怎么办？

##代理模式解决问题

info 具名发送转代理，代理匿名接收，就这样完成任务

code.js2

app1 = new App('app1');
app2 = new App('app2');

function App(){
    this.icon.click(function(){
        this.fire('open'); //实名发送
    });
}

App.prototype.fire = function(){ //实名转代理
    App.fire.apply(this, arguments); //使用代理匿名发送消息
};

//扩展模块使用代理监听消息，从而不需要实名一个个写事件监听
App.on('open', function(){
    //this指向发起消息的对象
})

.
