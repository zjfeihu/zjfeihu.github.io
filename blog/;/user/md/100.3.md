WEB++开发之模块设计

##前言

WEB++桌面系统中，使用模块化方式开发，模块即通过接口的方式向外提供特点功能的代码组合，如何组织模块将影响代码的可读性和程序的性能。

##友好的命名

+ 常量使用纯大写，使用下划线分隔单词
+ 名词形式的变量采用纯小写
+ 判断形式的变量采用驼峰
+ 表示状态的变量采用纯小写
+ 动名词形式的变量采用驼峰
+ 模块名使用首字母大写


##清晰的结构

+ 所有依赖模块在头部声明
+ 模块在顶部导出
+ 所有常量和变量在头部声明
+ 内部函数在底部声明
+ 尽早的跳出函数
+ 尽可能的减少嵌套层次

##合理的封装

+ 单例或者极少例可采用私有方式封装接口，即各种私有变量在闭包中声明和使用
+ 类模块使用原型方式向外开放接口
